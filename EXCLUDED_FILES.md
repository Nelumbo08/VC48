# Files Excluded from Repository

The following files and directories are present locally but excluded from version control via `.gitignore`:

## Directories

| Directory | Description |
|---|---|
| `__pycache__/` | Python bytecode cache (auto-generated, not source code) |
| `runs/` | TensorBoard experiment logs (54 training runs from Jan 2025). These contain binary event files and are too large/ephemeral for version control. |
| `checkpoints/vc1/` | Model checkpoint directory for saving trained model weights. Currently empty. |
| `.claude/` | Claude Code session metadata. |

## Files

| File | Description |
|---|---|
| `=0.2.0` | Output log from installing `torch-tb-profiler==0.4.3`. Not source code. |

## Rationale

- **Training artifacts** (`runs/`, `checkpoints/`) are excluded because they are large binary files that change frequently and can be regenerated by running the training pipeline.
- **Cache files** (`__pycache__/`) are auto-generated by Python and should never be committed.
- **Tool metadata** (`.claude/`) is user-specific session data.
- **Installation logs** (`=0.2.0`) are not part of the project source.
