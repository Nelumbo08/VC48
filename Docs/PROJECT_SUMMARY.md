# VC48 -- Project Summary

Complete summary of the Voice Conversion 48 project: what's in the repository, what's excluded, and full experiment results.

---

## Part 1: Files Uploaded to GitHub

The following source files are tracked in the repository:

| File | Description |
|---|---|
| `train.py` | Main training loop -- adversarial training with Generator and Discriminator over 1,000 epochs, batch size 32, Adam optimizer (lr=0.0001). Logs to TensorBoard every 100 steps. |
| `models.py` | Model architectures -- UNet-style Generator (encoder-decoder with skip connections), multi-period Discriminator with capsule network and attention-based routing, and iSTFT vocoder for waveform reconstruction. |
| `dataset.py` | Data loading -- `WaveformDataset` class that loads WAV files, resamples to 48 kHz, splits into 0.5s chunks, and computes 128-bin mel spectrograms on-the-fly. Also includes `Wav2Mel`, `LogMelspectrogram`, and `SoxEffects` utilities. |
| `layers.py` | Custom neural network layers -- capsule layers with attention routing, conformer blocks, depthwise separable convolutions, NMF layers, sub-pixel upsampling, gated linear units, HypSnake activation, and multi-head self-attention. |
| `utils.py` | Utilities -- loss functions (adversarial, perceptual, cycle, identity, style, F0, STFT, cosine, KL divergence, SSIM, MCD), audio processing, checkpoint save/load, and vocoder loading. |
| `test_audio.py` | Test script for mel spectrogram pipeline on GPU. |
| `test_cuda.py` | Test script for CUDA/GPU availability. |
| `Voice_conversion.md` | Architecture specification document. |
| `Docs/plan.txt` | Step-by-step implementation plan covering data prep, generator/discriminator design, loss functions, and training strategy. |
| `requirements.txt` | Python dependencies: torch, torchaudio, torchinfo, tensorboard, numpy, scipy, matplotlib. |
| `.gitignore` | Git ignore rules for excluding artifacts. |

### Two Distinct Training Phases

#### Phase 1: Full Loss Function (Jan 7 evening -- Jan 8 morning)

- **Experiments**: `20250107_185201` through `20250108_120918`
- **All 8 losses active** including cosine and STFT
- **Problem**: The cosine distance loss produced values around 768,000, completely overwhelming the other losses (which were in the 0--10 range)
- **Result**: Generator loss extremely high (7,600 -- 768,000), training unstable

#### Phase 2: Refactored Loss Function (Jan 8 afternoon)

- **Experiments**: `20250108_144850` through `20250108_152755`
- **4 losses only**: generator, discriminator, cycle, identity
- **Fix**: Removed cosine and STFT losses from the generator objective
- **Result**: Generator loss near zero or negative (meaning the generator fools the discriminator), training stable and converging

### Star Experiment: `experiment_20250108_152755`

The longest and best-performing run -- 94,100 training steps over ~19 hours of wall-clock time.

#### Learning Curve (sampled at 0%, 25%, 50%, 75%, 100%)

| Metric | Start | 25% | 50% | 75% | Final |
|---|---|---|---|---|---|
| Loss/generator | 1.3881 | -0.5854 | -0.7298 | -0.7436 | **-0.7455** |
| Loss/discriminator | 0.9993 | 0.7500 | 0.7500 | 0.7500 | **0.7500** |
| Loss/cycle | 0.5788 | 0.0507 | 0.0064 | 0.0022 | **0.0017** |
| Loss/identity | 0.7877 | 0.1139 | 0.0138 | 0.0041 | **0.0026** |

#### Analysis

- **Generator**: Loss went from +1.39 to -0.75. Negative generator loss means it consistently produces outputs the discriminator classifies as real.
- **Discriminator**: Stabilized at 0.75, indicating GAN equilibrium -- the discriminator is roughly 50/50 on distinguishing real from fake. This is the desired behavior.
- **Cycle consistency**: Dropped by **334x** (0.579 to 0.0017). The model preserves linguistic content through the A->B->A conversion round-trip.
- **Identity**: Dropped by **305x** (0.788 to 0.0026). When given target-speaker audio, the generator learns to reproduce it unchanged.
- **Convergence**: Clear and steady with no signs of mode collapse or oscillation.
